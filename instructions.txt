r.in.emit is a GRASS GIS Addons module, modeled directly on $HOME/dev/r.in.sentinel (see $HOME/dev/grass for the GRASS GIS source tree, $HOME/dev/grass-addons for the Addons source tree). It downloads NASA EMIT L2A Estimated Surface Reflectance granules by querying NASA's CMR (Common Metadata Repository) and fetching the matching reflectance NetCDF files directly from LP DAAC (a free NASA Earthdata account and Bearer token are required) — unlike r.in.sentinel's cubo-based exact bounding-box cube requests, EMIT granules are downloaded whole/full, as delivered, not cropped.

The module reads the current computational region (g.region -b, WGS84-reprojected) to determine the CMR search bounding box. By default, downloaded (and, with -i, imported) data goes into the current GRASS project/mapset the module was launched from; optionally, project=/dbase=/epsg= create (or reuse) a separate GRASS project instead.

-i drives i.hyper.import (product=emit) on each downloaded granule to produce a raster_3d hyperspectral cube, with a real acquisition timestamp set via r3.timestamp (not just the calendar date, since EMIT can overpass an area more than once per day). strds= (requires -i) registers all cubes imported in one run into a str3ds (Space-Time 3D Raster Dataset) via t.create/t.register.

Full complement of GRASS GIS Addons files: Makefile, CMakeLists.txt, README.md, LICENSE, r.in.emit.md, testsuite/test_r_in_emit.py. The testsuite's fast tests (bounding box resolution, CMR search/list) run without a token; real download/import tests are opt-in (RUN_EMIT_DOWNLOAD_TEST=1) since EMIT granules are large (hundreds of MB to a few GB each), unlike r.in.sentinel's tiny cubo-cropped test cube.
